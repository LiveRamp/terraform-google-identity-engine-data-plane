## Commands

```bash
# Initialize Terraform (required after cloning or adding modules)
terraform init

# Validate configuration
terraform validate

# Format all .tf files
terraform fmt -recursive

# Preview changes
terraform plan

# Apply changes
terraform apply

# Generate README.md from Terraform documentation (requires Docker)
docker run --rm --volume "$(pwd):/terraform-docs" -u $(id -u) \
  quay.io/terraform-docs/terraform-docs:0.17.0 markdown /terraform-docs > README.md
```

**Do not edit README.md manually** — it is auto-generated by the Jenkins CI pipeline using terraform-docs on every push to `main`.

## Architecture

This is a **Terraform module** that provisions GCP infrastructure for the **Identity Engine Data Plane** — a multi-tenant, security-hardened environment for customer data storage and processing at LiveRamp.

### What it provisions

| Area | Resources |
|------|-----------|
| Networking | VPC, subnetwork, Cloud Router, Cloud NAT (2 static IPs), firewall rules |
| Storage | GCS bucket, BigQuery dataset |
| Security | KMS key ring + crypto keys, service accounts |
| Access control | IAM bindings for `data_viewers` and `data_editors` (users/groups/SAs) |
| Compute | Dataproc-ready network configuration and firewall rules |

### Key patterns

**IAM member prefixing** (`locals.tf`): The `data_viewers` and `data_editors` variables accept plain email addresses for `users`, `groups`, and `service_accounts`. `locals.tf` builds the correctly prefixed member lists (`user:`, `group:`, `serviceAccount:`) consumed throughout the IAM resources.

**Encryption toggle**: `enable_encryption` (default `true`) gates all KMS key creation and CMEK configuration on buckets and BigQuery. When disabled, resources use Google-managed encryption.

**Dataproc network toggle**: `enable_dataproc_network` (default `true`) controls whether the VPC and subnet are created by this module. Can be disabled when networking is managed externally.

### `_modules/graph-visualiser/`

A local submodule that deploys the Identity Graph Visualiser — a Cloud Run v2 service protected by Identity-Aware Proxy (IAP). It is provisioned within the same data plane project and connects to the BigQuery knowledge base dataset. Access is controlled via IAP IAM bindings for authorized users and groups.

### CI/CD

Jenkins pipeline (see `Jenkinsfile`) runs on push to `main`:
1. Detects if the commit is not already a Jenkins commit
2. Runs `terraform-docs` via Docker to regenerate `README.md`
3. Commits and pushes the updated docs as `svc-jenkins`
